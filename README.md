# 🍕 Pizza Sales Analytics using SQL & Tableau

## 📖 Project Overview

This project is an end-to-end Data Analytics project built using **SQL** and **Tableau** to analyze pizza sales data and generate business insights.

The same dataset (`pizza_sales.csv`) was used in two different stages of the analytics workflow.

- **SQL** was used to query, aggregate, and analyze the raw sales data by answering business questions through SQL queries.
- **Tableau** was used to transform those analytical results into interactive dashboards and visualizations for easier business decision-making.

This demonstrates a complete Data Analyst workflow from raw data to business insights.

---

## 🎯 Problem Statement

A pizza restaurant wants to understand its sales performance by answering key business questions such as:

- What is the total revenue?
- How many orders were placed?
- Which pizzas generate the highest revenue?
- Which pizzas perform poorly?
- Which hours are the busiest?
- Which pizza category contributes the most sales?
- Which pizza size is the most popular?

The objective is to analyze the sales data and present meaningful insights through an interactive dashboard.

## 💡 Why SQL and Tableau?

Although the same dataset was used, each tool served a different purpose.

### SQL

SQL was used to:

- Import the dataset into SQL Server.
- Perform data analysis.
- Calculate KPIs.
- Write business queries.
- Aggregate and summarize sales data.

Examples include:

- Total Revenue
- Total Orders
- Average Order Value
- Hourly Sales Trend
- Weekly Orders
- Top & Bottom Selling Pizzas
- Sales by Category
- Sales by Pizza Size

---

### Tableau

Tableau was used to:

- Build interactive dashboards.
- Create KPI cards.
- Visualize business trends.
- Develop charts and graphs.
- Add filters for user interaction.
- Present business insights in a visual format.
